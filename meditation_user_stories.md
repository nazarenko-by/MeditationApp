# Meditation User Stories

## Exercise 2: Login/Registration Page

| Title | Description | Acceptance Criteria | Story Points |
|-------|-------------|---------------------|--------------|
| Sign Up | As a user, I want to sign up by providing a username, email, and password, so I can create an account. | 1. Users can enter valid details and click “Sign Up” to register.<br>2. Error messages appear if any field is invalid or left empty. | 3 |
| Log In | As a user, I want to log in with my email and password, so I can access my account. | 1. Users can log in with valid credentials and are redirected to the dashboard.<br>2. Invalid credentials trigger an error message. | 3 |
| Empty Fields Feedback | As a user, I want to receive feedback when trying to sign up or log in without filling all required fields. | 1. Show clear error messages when any required input is missing. | 2 |
| Local Storage | As a user, I want my login and registration data saved in local storage. | 1. After sign-up, user data is saved locally and used for login authentication. | 3 |

## Exercise 5: Favorites Feature

| Title | Description | Acceptance Criteria | Story Points |
|-------|-------------|---------------------|--------------|
| Add to Favorites | As a user, I want to mark content as a favorite. | 1. Display a heart icon with “Add to Favorites.”<br>2. Empty heart shows it’s not saved.<br>3. Tapping saves the item, changes text and icon. | 4 |
| Remove from Favorites | As a user, I want to remove items from my favorites. | 1. “Remove from Favorites” is shown for saved items.<br>2. Clicking removes item and reverts icon.<br>3. Toggle anytime. | 3 |
| Favorites Screen | As a user, I want a dedicated screen for my favorites. | 1. Display a list with title, category, and duration.<br>2. Users can tap items to view or start.<br>3. Ensure list is organized. | 5 |
